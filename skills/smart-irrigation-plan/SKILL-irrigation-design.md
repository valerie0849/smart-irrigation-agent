# 灌溉设计技能 — 灌溉制度设计与需水量计算

## 何时使用

当需求分析引擎返回项目概况后，调用灌溉设计引擎设计灌溉制度。

## 行动指南

### 第1步：知识检索

调用知识森林检索设计相关知识，查询关键词包含：作物类型、地点、灌溉制度、灌水定额、Kc系数、设计方案。返回最多5条结果。

### 第2步：计算参考作物需水量 ET0

使用 FAO-56 Penman-Monteith公式 计算ET0，输入气象数据（温度、湿度、风速、日照时数），按以下顺序计算：

1. 计算饱和水汽压斜率（delta）
2. 计算干湿表常数（gamma）
3. 计算大气顶层辐射（ra）
4. 估算太阳辐射（rs = 0.5 × ra）
5. 计算净辐射（rn = 0.77 × rs）
6. 土壤热通量（日尺度近似为0）
7. 代入Penman-Monteith公式求解ET0

计算结果取 max(ET0, 1.0)，确保最小值为1.0 mm/d。异常时返回默认值3.5 mm/d。

### 第3步：Kc系数验证

从知识检索结果中查找Kc相关记录。仅当找到≥2条Kc相关记录时，添加知识验证标记。Kc系数以数据库/默认值为准，知识检索结果仅用于验证背书，不修改数值。

### 第4步：生成灌溉制度

为4个生育期（苗期、分蘖期、拔节期、成熟期）分别计算：

1. **作物需水量**：ETc = ET0 × Kc
2. **阶段需水量**：需水量 × 生育期天数
3. **灌水定额**：有效含水量 × 0.6（效率系数）
4. **灌水周期**：向上取整(有效含水量 × 0.6 / ETc)
5. **灌水次数**：向上取整(生育期天数 / 灌水周期)

### 第5步：选择灌溉方式

根据土壤类型和知识推荐选择灌溉方式：
- 砂土 → 默认滴灌
- 粘土 → 默认沟灌
- 壤土 → 默认喷灌

知识森林中"滴灌"出现频次大于"喷灌" → 选择滴灌，反之选喷灌。

### 第6步：构建结果

返回设计结果字典，包含：项目信息（地点/作物/面积/总需水量/灌溉方式/知识支撑标记）、灌溉制度（4个生育期详细数据）、计算细节（ET0/Kc值/土壤参数/知识来源）、知识学习内容。

## 决策标准

| 场景 | 决策 |
|-----|------|
| 知识森林有Kc相关记录≥2条 | 保留数据库Kc值，添加知识验证标记 |
| 知识森林无Kc记录或查询失败 | 仅使用数据库/默认Kc值 |
| 气象数据缺失（异常时） | 使用默认ET0=3.5 mm/d |
| Penman-Monteith计算异常 | 返回默认3.5 |
| 砂土 + 知识推荐滴灌 | 选择滴灌 |
| 粘土 + 无知识推荐 | 选择沟灌 |
| 壤土 + 无知识推荐 | 选择喷灌 |

## 质量检查

在输出前确认：
- ET0 ≥ 1.0 mm/d
- 4个生育期全部覆盖（苗期/分蘖期/拔节期/成熟期）
- Kc值呈先升后降趋势（拔节期最高）
- 灌水周期 ≥ 1天
- 灌水定额 = 有效含水量 × 0.6
- 灌溉方式与土壤类型匹配
- 总需水量计算包含面积换算（mm × 亩 × 666.67 / 1000 = m³）
