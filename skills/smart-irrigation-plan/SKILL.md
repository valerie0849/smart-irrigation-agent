# 智慧水利灌溉方案智能生成系统

## 技能概述

这是一个多引擎协作的灌溉方案智能生成技能集，由7个引擎和3个支撑模块组成，覆盖从需求输入到方案输出的完整业务流程。技能采用调度引擎统一路由、需求分析引擎提取实体、灌溉设计引擎计算需水量、设备选型引擎配置设备、编排引擎生成报告、质检引擎审核质量、修改引擎支持局部修改。

## 何时使用

| 触发场景 | 触发关键词 |
|---------|-----------|
| 方案生成 | 生成方案、设计方案、制定方案、出方案、帮我设计 |
| 方案修改 | 修改方案、调整预算、改成、改一下、换成 |
| 环境查询 | 天气、气象、气温、降水、降雨、湿度、土壤、土质 |
| 知识检索 | 知识、依据、标准、规范、参考标准 |
| 知识浏览 | 查看知识森林、浏览知识库、知识全貌 |

## 技能架构

系统分为四层结构：

**交互展示层**：用户界面与交互入口，连接调度引擎

**智能决策层**：核心业务引擎群
- 调度引擎：意图分类、实体提取、流程路由
- 需求分析引擎：项目实体提取、环境数据采集
- 灌溉设计引擎：ET0计算、Kc验证、灌溉制度生成
- 设备选型引擎：设备清单匹配、预算优化
- 方案编排引擎：多源知识合并、Markdown报告生成
- 质检引擎：四维质量评估、改进建议生成
- 方案修改引擎：定向修改、依赖展开、质量迭代

**数据中台层**
- MySQL/SQLite：存储作物参数、土壤参数、知识节点
- Faiss向量库：知识森林索引，支持向量检索

**基础设施层**
- Open-Meteo API：提供气象和土壤实时数据
- DeepSeek LLM：API模式大语言模型
- 领域知识库：PDF文献、标准规范

## 核心工作流

**方案生成流程**：用户输入 → 调度引擎（意图分类→实体提取→缺失信息检测）→ 需求分析引擎（实体提取→坐标解析→气象API→土壤API→DB查询）→ 灌溉设计引擎（知识检索→ET0计算→Kc验证→灌溉制度→方式选择）→ 设备选型引擎（知识检索→设备清单→知识增强→预算优化）→ 方案编排引擎（知识合并→模板渲染→方案说明）→ 质检引擎（合规性→合理性→充分性→知识基准→加权评分）→ 输出方案

**方案修改流程**：用户修改指令 → 调度引擎（检测历史方案+参数微调→转为modify意图）→ 方案修改引擎（LLM解析意图→BFS依赖展开→并行检索→骨架生成→合并→局部审核→质量迭代）→ 输出修改后方案

## 技术栈

| 组件 | 技术 |
|-----|------|
| LLM | DeepSeek-V4-Pro (API模式) |
| 气象API | Open-Meteo (current参数获取实时数据) |
| 土壤API | Open-Meteo (hourly参数) |
| 向量库 | Faiss IndexIDMap (IndexFlatL2, dim=256) |
| 向量化 | TF-IDF(max_features=5000) + SVD(n_components=256) |
| 聚类 | GaussianMixture (sklearn) |
| 关系库 | SQLite/MySQL |
| 模板引擎 | Jinja2 |

## 质量保障

所有输出方案必须通过四维质量审核：

| 维度 | 权重 | 核心检查项 |
|-----|------|-----------|
| 合规性 | 35% | 灌溉水利用系数≥0.6、灌水定额≤80m³/亩、标准引用 |
| 合理性 | 30% | Penman-Monteith(20分)、FAO(15分)、Kc(20分)、生育期(15分)、定额(15分)、频次(15分) |
| 充分性 | 20% | 项目概况/灌溉制度/设备/说明四章节 |
| 知识基准 | 15% | standard tag检索(15分)、template tag检索(10分)、利用系数(10分)、定额(5分)、标准编号(5分) |

评分 ≥ 90: 优秀；75-89: 良好；60-74: 合格；< 60: 需改进
