# 质检技能 — 四维质量审核

## 何时使用

当方案编排引擎完成报告生成后，调用质检引擎进行审核。

## 行动指南

### 第1步：合规性检查（权重35%）

初始100分，逐项扣分：

1. **灌溉水利用系数检查**：正则匹配方案文本
   - 小于0.6 → 扣20分（不符合国标要求）
   - 大于等于0.6 → 通过
   - 未找到 → 扣15分

2. **灌水定额检查**：正则匹配方案文本
   - 大于80 m³/亩 → 扣10分
   - 小于等于80 → 通过

3. **需水量计算检查**：关键词匹配（ET0/需水量）
   - 包含 → 通过

4. **标准规范引用检查**：关键词匹配（GB/SL/FAO）
   - 包含 → 通过

### 第2步：合理性检查（权重30%）

加分制（满分100）：

- 采用Penman-Monteith公式计算ET0 → 加20分
- 参考FAO-56标准 → 加15分
- 考虑作物系数Kc → 加20分
- 分生育期制定灌溉方案 → 加15分
- 包含灌水定额计算 → 加15分
- 包含灌水频次安排 → 加15分

面积小于100亩 → 额外扣10分（灌溉方案可能不经济）

### 第3步：充分性检查（权重20%）

按4个核心章节存在性计分（每章25分）：

- 项目概况 → 加25分
- 灌溉制度设计 → 加25分
- 设备选型清单 → 加25分
- 方案说明与依据 → 加25分

### 第4步：知识库基准检查（权重15%）

初始100分，缺失扣分：

1. 从知识森林检索标准规范（standard标签），找不到 → 扣15分
2. 检索方案模板（template标签），找不到 → 扣10分
3. 方案包含灌溉水利用系数 → 通过，否则扣10分
4. 方案包含灌水定额 → 通过，否则扣5分
5. 方案明确引用标准编号（GB/SL/FAO）→ 通过，否则扣5分

**知识森林不可用时**：返回None，上层自动调整权重（合规性40%+合理性35%+充分性25%）。

### 第5步：加权计算

综合得分 = 合规性 × 35% + 合理性 × 30% + 充分性 × 20% + 知识基准 × 15%

### 第6步：生成改进建议

从各维度检查项中提取带有警告标记（[!]）的内容，汇总为改进建议列表。无问题时提示"方案整体质量良好，可直接使用"。

## 决策标准

| 总分 | 等级 | 处理策略 |
|-----|------|---------|
| ≥ 90 | 优秀 | 直接输出 |
| 75-89 | 良好 | 直接输出 |
| 60-74 | 合格 | 直接输出 |
| < 60 | 需改进 | 自动触发重新生成 |

| 场景 | 决策 |
|-----|------|
| 知识森林不可用 | 跳过知识基准检查，权重调整为合规性40%+合理性35%+充分性25% |
| 灌溉水利用系数 < 0.6 | 合规性扣20分 |
| 灌水定额 > 80 | 合规性扣10分 |
| 面积 < 100亩 | 合理性扣10分 |
| 缺少章节 | 充分性按缺失章节数扣分（每缺1章扣25分） |

## 资源文件

| 资源 | 说明 |
|-----|------|
| re.search() | 本地正则匹配，输入方案文本，输出参数值匹配 |
| KnowledgeForest.tagged_query() | async调用，输入查询、标签和k，输出带标签的知识结果 |

## 质量检查

在输出前确认：
- 四个维度全部执行完毕（知识基准可跳过返回None）
- 分数在 0~100 范围内
- 等级与分数匹配正确
- 改进建议来源于实际的警告标记
- 权重总和为 1.0
- 改进建议列表不为空

## 输出规范

审核结果包含：合规性评分和明细、合理性评分和明细、充分性评分和明细、知识基准评分和明细、综合得分、等级、改进建议列表。
