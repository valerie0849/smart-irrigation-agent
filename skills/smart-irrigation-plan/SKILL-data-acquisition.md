# 数据采集技能 — Open-Meteo API封装与坐标解析

## 何时使用

当需求分析引擎需要获取项目地点的气象和土壤数据时调用。也支持用户直接查询天气或土壤信息。

## 行动指南

### 第1步：坐标解析

从覆盖150+城市/区县的坐标字典中匹配地点坐标：

1. **精确匹配**：地点名称完全匹配 → 直接返回坐标
2. **模糊匹配**：地点名称包含字典key → 取最长匹配的坐标
3. **默认回退**：无匹配 → 返回北京坐标

### 第2步：获取气象数据

请求 Open-Meteo API 的 current 参数获取实时气象数据：

- **请求参数**：纬度、经度、current字段（气温/湿度/风速/ET0）、时区
- **返回数据**：
  - 2米气温（℃）
  - 2米相对湿度（%）
  - 10米风速（m/s）
  - FAO参考作物蒸散发量 ET0（mm/天）
  - 数据时间戳
  - 数据源标注："Open-Meteo API (实时)"

API调用失败时返回默认值：温度25℃、湿度60%、风速2m/s、ET0 3.5mm/d。

### 第3步：获取土壤数据

请求 Open-Meteo API 的 hourly 参数获取土壤数据：

- **请求参数**：纬度、经度、hourly字段（三档深度土壤温度和湿度）、预报天数1天、时区
- **处理逻辑**：
  - 计算24小时平均土壤温度（0cm深度）
  - 计算三层土壤湿度加权平均（0-1cm + 1-3cm + 3-9cm）
  - 根据湿度推断土壤类型
  - 查询对应土壤类型的田间持水量、萎蔫系数、有效含水量

**土壤类型推断规则**：
- 湿度 > 0.4 → 粘土
- 湿度 < 0.2 → 砂土
- 0.2 ≤ 湿度 ≤ 0.3 → 粉壤土
- 0.3 < 湿度 ≤ 0.4 → 壤土

API调用失败时返回默认值：壤土、温度15℃、持水量0.32、萎蔫系数0.12、有效含水量0.20。

## 决策标准

| 场景 | 决策 |
|-----|------|
| 精确匹配城市坐标 | 直接返回坐标 |
| 模糊匹配（地点包含字典key） | 取最长匹配的坐标 |
| 无匹配 | 默认北京坐标 |
| Open-Meteo API调用失败 | 返回默认气象/土壤值 |
| 土壤湿度异常（无数据） | 默认0.3 |

## 资源文件

| 资源 | 说明 |
|-----|------|
| Open-Meteo API (气象) | HTTP GET请求，输入纬经度和current参数，输出JSON响应（温度/湿度/风速/ET0） |
| Open-Meteo API (土壤) | HTTP GET请求，输入纬经度和hourly参数，输出JSON响应（土壤温度/湿度） |
| 城市坐标字典 | 本地查找，输入地点字符串，输出纬度和经度 |

## 质量检查

在返回数据前确认：
- 气温在合理范围内（由API保证）
- 湿度在 0~100% 范围内
- 风速 ≥ 0
- ET0 > 0
- 土壤类型为有效值（粘土/壤土/砂土/粉壤土）
- 田间持水量 > 萎蔫系数（物理约束）
- 有效含水量 = 田间持水量 - 萎蔫系数
- 数据源标注清晰（实时API / 默认值 / 错误降级）

## 输出规范

**气象数据**：地点、经纬度、气温、相对湿度、风速、ET0、时间戳、数据源
**土壤数据**：地点、经纬度、土壤类型、土壤温度、表层湿度、田间持水量、萎蔫系数、有效含水量、数据源
