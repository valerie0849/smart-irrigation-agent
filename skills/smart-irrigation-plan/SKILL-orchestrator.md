# 方案编排技能 — 结构化报告生成

## 何时使用

当灌溉设计、设备选型完成，且多通道知识检索就绪后，调用方案编排引擎生成报告。

## 行动指南

### 第1步：知识合并

合并三个来源的知识引用并去重，按设计→通用→设备的顺序合并：

- **设计知识**：灌溉设计引擎的知识检索结果
- **通用知识**：调度层多通道检索（模板/技术/标准三个通道）
- **设备知识**：设备选型引擎的知识检索结果

**去重策略**：使用来源+内容前80字符作为唯一标识，相同内容只保留首次出现的条目。

### 第2步：Jinja2模板渲染

使用 Jinja2 模板引擎渲染五章Markdown报告：

**第一章 项目概况**：
- 项目基本信息（地点、作物、面积、预算）
- 气象条件（实时数据标注来源）
- 土壤条件（实时数据标注来源）

**第二章 灌溉制度设计**：
- 设计参数（计算方法、ET0、灌溉方式、总需水量、灌溉水利用系数）
- 生育期灌溉方案（Markdown表格：生育阶段/天数/Kc/日均需水/阶段需水/灌水定额/灌水次数）

**第三章 设备选型清单**：
- 设备表格（设备名称/型号/数量/单价/小计）
- 设备总投资

**第四章 溯源依据**：
- 知识森林参考来源（显示来源/相似度/内容摘要）
- 参考标准与文献（有来源数据用来源，无则用4个默认标准：FAO-56、GB/T 50363-2018、GB/T 50085-2017、SL/T 699-2025）

**第五章 方案说明**：
- 固定文本模板总结方案核心内容

### 第3步：生成方案说明

自动生成方案摘要文字，包含：
- 项目地点、作物、面积
- 参考作物需水量ET0
- 各生育期天数/Kc/需水量
- 推荐灌溉方式
- 参考知识源（最多取3个）

### 第4步：构建输出

返回编排结果字典，包含：方案内容（Markdown）、方案说明文字、标准来源列表。

## 决策标准

| 场景 | 决策 |
|-----|------|
| 知识森林无检索结果 | 知识参考显示"暂无知识森林参考来源" |
| 标准来源为空 | 使用4个默认标准 |
| 作物为推断值 | 作物类型后显示"（根据地区自动推断）" |
| 数据字段缺失 | 使用默认空值或"--" |
| 知识参考为字符串而非字典 | 模板分支处理 |

## 资源文件

| 资源 | 说明 |
|-----|------|
| Jinja2 Environment | 模板渲染，输入模板字符串和13个变量，输出Markdown文本 |
| 数字格式化 | Python内置函数，输入金额/比例，输出千分位/百分比字符串 |

## 质量检查

在输出前确认：
- 报告包含全部5个章节
- 气象数据标注了来源
- 土壤数据标注了来源
- 设备清单为Markdown表格格式
- 知识参考来源已列出
- 方案说明文字生成了完整摘要
- 预算格式化为千分位两位小数
