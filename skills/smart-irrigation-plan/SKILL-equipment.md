# 设备选型技能 — 设备配置与预算优化

## 何时使用

当灌溉设计引擎返回灌溉方案后，调用设备选型引擎配置设备。

## 行动指南

### 第1步：知识检索

调用知识森林检索设备配置相关知识，查询关键词包含：灌溉方式、作物类型、地点、设备选型、配置方案。返回最多3条结果。

### 第2步：生成设备清单

根据灌溉方式选择预设设备模板，按项目面积计算设备数量：

**滴灌设备清单**：
- 滴灌带：面积 × 1.2 米
- 滴灌管：面积 × 0.1 米
- 电磁阀：面积 ÷ 50 + 1 个
- 过滤器：2台
- 水肥一体机：1台
- 传感器：面积 ÷ 100 + 1 个
- 控制器：1台

**喷灌设备清单**：
- 喷头：面积 ÷ 0.15 个
- 喷灌管：面积 × 0.15 米
- 电磁阀：面积 ÷ 100 + 1 个
- 过滤器：2台
- 水泵：2台
- 传感器：面积 ÷ 100 + 1 个
- 控制器：1台

**沟灌设备清单**：
- 渠道衬砌：面积 × 0.05 米
- 闸门：5个
- 流量计：3台
- 水泵：2台

### 第3步：知识增强

检测知识检索内容中的关键词，增强设备配置建议：

- 检测到IoT相关词（物联网/IoT/传感器网络/智能监测）且已配置控制器 → 标注智能控制建议
- 检测到自动化关键词（自动控制/智能控制/远程/Docker/Kubernetes）且已配置控制器 → 标注智能控制建议
- 设备名称出现在知识内容中 → 标记知识引用

### 第4步：预算优化

计算设备总成本，超出预算时执行两级优化：

**第1级优化（ratio < 0.5，预算严重不足）**：
- 移除非关键设备：传感器、水肥一体机、控制器

**第2级优化（0.5 ≤ ratio < 1.0，预算略不足）**：
- 关键设备（过滤器、水泵）保持原数量
- 其他设备按比例缩减，但至少保留1个

### 第5步：构建结果

返回设备选型结果字典，包含：灌溉方式、设备清单、总成本、预算、剩余预算、知识来源。

## 决策标准

| 场景 | 决策 |
|-----|------|
| 检测到IoT关键词且已配置控制器 | 添加智能控制建议备注 |
| 检测到自动化关键词且已配置控制器 | 添加智能控制建议备注 |
| 预算比例 < 0.5 | 移除传感器/水肥一体机/控制器 |
| 预算比例 0.5~1.0 | 关键设备保持，其他按比例缩减 |
| 预算比例 ≥ 1.0 | 不调用优化，保持原清单 |
| 知识森林不可用 | 跳过增强，使用默认配置 |
| 灌溉方式未知 | 默认使用喷灌配置 |

## 资源文件

| 资源 | 说明 |
|-----|------|
| KnowledgeForest.query() | async调用，输入查询字符串和k=3，输出设备知识结果 |
| 本地计算 | 按公式计算，输入面积和灌溉方式，输出设备数量 |

## 质量检查

在输出前确认：
- 每种设备包含名称/型号/数量/单价/单位字段
- 关键设备（过滤器/水泵）数量未被缩减
- 每种设备至少保留1个
- IoT/自动化建议已标注（当知识包含相关关键词时）
- 知识参考设备已标记
- 总成本 ≤ 预算（优化后）
