# 知识森林技能 — 文档解析、向量化、聚类、检索

## 何时使用

- 系统启动时：尝试从缓存加载
- 知识森林未构建时：触发完整构建
- 方案生成流程中：按需检索
- 用户请求浏览知识库时：获取知识全貌

## 领域知识

### 三层知识结构

**L0（文本块层）**：
- 文档切分后的原始文本片段
- 由文本分割器切分
- 作为检索的最细粒度层

**L1（单文档摘要层）**：
- 每个文档内GMM聚类生成的摘要
- 保留文档内语义结构
- 作为中层知识节点

**L2/L3（跨文档摘要层）**：
- 跨文档GMM聚类生成的高层知识
- L2由L1摘要递归聚类生成
- 当L2摘要数大于5时，再生成L3
- 作为最高层知识导航

### 管道9阶段

知识森林构建通过9阶段管道自动完成：

1. **文档加载**：扫描PDF/TXT文件，过滤临时文件，解析文本和表格
2. **标签分类**：关键词匹配打分，将文档归入三类（template结构模板、standard合规标尺、domain_knowledge技术参数）
3. **文本分割**：按段落边界和标题层级切分，512字符阈值，生成L0文本块
4. **向量化**：TF-IDF提取特征（max_features=5000）+ SVD降维至256维
5. **单文档聚类摘要**：GMM软聚类（n_clusters=min(3, 块数÷3)），并行处理生成L1摘要
6. **跨文档聚类摘要**：L1→L2递归聚类，L2>5时再生成L3
7. **标签传播**：计算来源文档标签比例，确定主导标签
8. **FAISS索引构建**：收集L0+L1+L2节点，IndexIDMap+FlatL2，保存索引文件
9. **数据库持久化**：清除旧记录，写入知识节点表，更新版本号

### 检索机制

**基础检索**：查询文本向量化后在FAISS索引中搜索最相似结果，计算相似度返回来源、内容和标签。

**标签过滤检索**：扩大检索范围后过滤指定标签类型，用于检索标准规范和方案模板。

**多通道并行检索**：同时发起模板、技术、标准三个通道并行检索，差异化融合：
- 模板通道：决定结构框架
- 技术通道：填充参数细节
- 标准通道：提供合规依据

### 增量更新机制

对比文件MD5指纹，仅处理新增或修改文件：
- 新旧文档比例 ≤ 20% → 执行增量更新
- 新旧文档比例 > 20% → 触发全量重建

## 决策标准

| 场景 | 决策 |
|-----|------|
| 知识库目录为空 | 跳过构建 |
| 文档解析失败 | 跳过该文档，继续处理后续文档 |
| L1摘要 ≤ 3 | 跳过跨文档聚类 |
| L2摘要 > 5 | 递归生成L3摘要 |
| 检索结果为空 | 返回空列表，上层使用默认知识 |

## 资源文件

| 资源 | 说明 |
|-----|------|
| DocumentParser.parse_document() | 本地实现，输入PDF/TXT文件路径，输出结构化文档列表 |
| TextSplitter.split_documents() | 本地实现，输入文档列表，输出文本块列表 |
| TextEmbedding.fit() + batch_encode() | 本地实现，输入文本列表，输出TF-IDF+SVD向量矩阵(256维) |
| GMM聚类 + 摘要生成 | 本地实现，输入文本块和向量，输出L1/L2/L3摘要列表 |
| faiss.IndexIDMap(IndexFlatL2) | 本地实现，输入向量数组，输出向量索引 |
| KnowledgeNode (数据库) | SQLAlchemy实现，输入节点数据，输出持久化记录 |

## 质量检查

构建完成后确认：
- L0文本块数量 > 0
- L1摘要数量 > 0
- FAISS索引维度 = 256
- 索引文件已保存
- 数据库记录已更新
- 系统版本号已更新
